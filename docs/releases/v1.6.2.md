## 🎯 Obiettivo

Risolvere il problema del ranking "qualità prezzo" che mostrava vini costosi (Bolgheri €180) invece di vini economici, e rendere l'app iOS compatibile con i nuovi campi backend.

---

## 🐛 Bug Risolti

### BUG-001: Query "qualità prezzo" mostra vini costosi
**Sintomo**: Query "Vino rosso qualità prezzo" restituiva Bolgheri DOC €180 invece di vini €11-16  
**Root Cause**: 
1. Peso `value_score` troppo basso (0.20) nel ranking
2. `value_intent` non incluso in `active_signals` → badge mai mostrato
3. Formula value permetteva vini costosi con qualità alta

**Fix Applicati**:
1. **Backend `main.py`**: Aggiunto `"value_intent": value_intent` a `active_signals` (riga ~2471)
2. **Backend pesi**: Wv=0.20→0.50 (aumentato peso value_score)
3. **UI helpers**: Badge solo per vini ≤€50 + qualità ≥0.75

**Commit**: `647252d` - "Fix show_value_badge: add value_intent to active_signals"

---

### BUG-002: iOS app crash su parsing SSE response
**Sintomo**: App iOS rimaneva appesa senza mostrare risultati dopo query  
**Root Cause**: 
1. Modello Swift `WineCard` mancava campi nuovi dal backend
2. `__components` aspettava `[String: Double]` ma backend inviava nested dictionaries
3. Cache HTTP iOS serviva risposte vecchie

**Fix Applicati**:
1. **iOS `Models.swift`**: 
   - Aggiunto `show_value_badge`, `sparkling`, `freshness`, `sweetness`
   - Aggiunto `aroma_icons`, `reviews_count`, `critic_score`, `match_explanation`
   - Decoder `__components` reso robusto con `try?` invece di `try`
2. **iOS `APIClient.swift`**: 
   - Cache HTTP disabilitata: `.cachePolicy = .reloadIgnoringLocalAndRemoteCacheData`
   - Applicato a POST `/search` e GET `/search_stream`

**File modificati**:
- `ios-app/SommelierAI/SommelierAI/Models.swift`
- `ios-app/SommelierAI/SommelierAI/APIClient.swift`
