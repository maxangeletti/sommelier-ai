## 🐛 Bugfixes

### Backend Search Keyword Filter
**Issue**: Query "bollinger" restituiva ENTRAMBI Bollinger (ID 52) E Louis Roederer (ID 53)

**Root Cause**:
1. `row.get()` non funziona con pandas `iterrows()` (restituisce Series, non dict)
2. Condizione `if len(keyword_matches) < len(filtered)` impediva filtro se match = 100%
3. Filtro opzionale invece di obbligatorio

**Fix Applied** (`backend/main.py` riga ~2450):
```python
# ✅ FIX: Keyword filter PRODUCER-first
keyword_matches = []
q_lower = q.lower().strip()

for idx, row in filtered.iterrows():
    # pandas iterrows() restituisce Series - usa indicizzazione diretta
    producer_lower = str(row['producer']).lower() if 'producer' in row.index else ''
    name_lower = str(row['name']).lower() if 'name' in row.index else ''
    
    if q_lower in producer_lower or q_lower in name_lower:
        keyword_matches.append(idx)

# Se ci sono match esatti su PRODUCER/NAME, mostra SOLO quelli
if len(keyword_matches) > 0:
    filtered = filtered.loc[keyword_matches]
```

**Changes**:
- `row.get('producer')` → `row['producer']` (corretto per pandas Series)
- Rimossa condizione `< len(filtered)` - ora filtra SEMPRE se trova match
- Aggiunto controllo `if 'producer' in row.index` per sicurezza

- ID 53: producer="Louis Roederer", name="Champagne AOC Collection"
